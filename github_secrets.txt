GitHub Secrets Setup Guide
==========================

Для работы CI/CD в GitHub Actions нужно создать следующие секреты:

1. DOCKER_USERNAME
   - Ваш логин на Docker Hub
   - Используется для деплоя Docker образов
   - Пример: fhvfutl

2. DOCKER_PASSWORD
   - Ваш пароль от Docker Hub (или Personal Access Token)
   - Рекомендуется создать PAT с правами write

3. GITHUB_TOKEN
   - Создается автоматически GitHub Actions
   - Не нужно настраивать вручную

4. ROUTERAI_API_KEY
   - Ваш API ключ от Routerai
   - sk-negv3Xex4r1h7F9i6_LxnLlx0vM5AG44

5. SSH_PRIVATE_KEY
   - Приватный SSH ключ для подключения к серверам
   - Создан: ~/.ssh/id_ed25519_github

6. SSH_PUBLIC_KEY
   - Публичный SSH ключ для authorized_keys
   - Создан: ~/.ssh/id_ed25519_github.pub

=== Как добавить секреты ===

1. Зайдите на GitHub: https://github.com/fhvfutl/homecompany-core
2. Перейдите в Settings → Secrets and variables → Actions
3. Нажмите "New repository secret"
4. Добавьте каждый секрет с названием и значением

=== Проверка ===

После добавления всех секретов, проверьте workflow:
- Создайте pull request
- Посмотрите на вкладку "Actions"
- Убедитесь, что все задачи (lint, test, deploy) проходят успешно

=== Доступные Secrets ===

- DOCKER_USERNAME
- DOCKER_PASSWORD
- ROUTERAI_API_KEY
- SSH_PRIVATE_KEY
- SSH_PUBLIC_KEY

=== Важно ===

1. DOCKER_PASSWORD: Лучше использовать Personal Access Token (PAT) вместо пароля
2. SSH_PRIVATE_KEY: Скопируйте полный ключ с -----BEGIN...----- и -----END...-----
3. Проверьте права доступа к репозиторию
4. Тестируйте workflow перед деплоем

=== Troubleshooting ===

Если workflow не запускается:
1. Проверьте наличие .github/workflows/ci.yml
2. Убедитесь, что все Secrets добавлены
3. Проверьте логи на GitHub Actions вкладке
4. Проверьте, что ветка называется 'main' (не 'master')

=== Next Steps ===

После настройки CI/CD:
1. Создайте pull request
2. Проверьте запуск workflow
3. Слияние PR запустит автоматический деплой
4. Проверьте статус deployment на GitHub Pages/Docker
